
# AI 交易智能体（TradingAgent）执行流程（多模态感知 → 逻辑推理 → 动态风控与指令下发）

把“海量市场异构数据与账户状态”如何一路变成“交易所 API 执行指令”讲清楚：账户初始化与约束 → 多源数据感知 → 状态编码与 Prompt 构造 → 核心推理层（CoT/多智能体）→ 信号解码与意图提取 → 刚性风控与置信度过滤 → 路由执行与状态更新。

---

## 1. 初始化与资金盘点：摸清家底，设好红线

1. **查余额（调接口）**：通过券商的 API（如 富途牛牛 OpenAPI、盈透 IBKR API 或 `akshare` 数据接口）查一下当前账户里还有多少可用资金（如 USD/CNY）、持有多少股票（为了告诉大模型“你现在有多少钱可以动”）。
2. **写死风控参数（硬编码）**：在配置文件（`.env` 或 `config.json`）里写死：单笔最多买 10%、亏损 5% 必须平仓、绝不开杠杆（为了防止大模型发神经把你的钱亏光，这部分**绝对不能**交给 AI 决定）。
3. **锁定自选股**：定义一个 List，比如 `['AAPL', 'NVDA', 'TSLA']`（美股）或 `['600519', '000858']`（A股），只让机器盯着这几个看（为了省 API 请求费和大模型的 Token 钱）。

---

## 2. 收集市场数据：给大模型喂“饲料”

1. **抓取 K 线数据（数值）**：调用接口获取最近的日K或分时价格（开盘/收盘/最高/最低价）和成交量，算一下简单的指标比如均线（MA）、RSI 等。
2. **抓取新闻情绪（文本）**：用爬虫或 API 抓一下推特上大 V 说了什么、雪球论坛在炒作什么、或者最新的财经新闻（如财联社、彭博社）（这是大模型最擅长处理的“黑天鹅”信息）。
3. **打包对齐**：把上面获取的“K线数字”和“新闻文字”按时间戳对齐，过滤掉没用的废话。

---

## 3. 拼装 Prompt：把数据塞进大模型

1. **组装长文本**：把刚才的 K 线和新闻，套进一个 Markdown 或 JSON 模板里。比如：“现在 NVDA（英伟达）价格是 120，RSI 是 70，最新新闻是美联储降息，请问接下来怎么操作？”
2. **加上历史记忆（可选）**：从本地数据库（如 SQLite 或向量库）里翻一下，上次遇到这种情况时我们是怎么操作的，亏了还是赚了，把经验也写进 Prompt 里。
3. **设定 System Prompt（洗脑）**：告诉大模型：“你现在是一个华尔街顶级交易员，你的任务是分析以上数据，并严格按照我要求的 JSON 格式输出你的交易决定。”

---

## 4. 大模型推理：让 AI 思考与辩论

1. **强制它写“思考过程”（CoT）**：在 Prompt 里要求大模型必须先输出 `<think>` 分析过程（比如列出看涨的理由和看跌的理由），然后再给出结论（这样做能大幅提高大模型的判断准确率，避免它瞎猜）。
2. **多 Agent 吵架模式（进阶玩法）**：如果觉得一个模型不靠谱，你可以开三个模型：一个专门看技术面，一个专门看消息面，最后一个做“经理”综合它俩的意见拍板（类似于真实公司的投研团队）。

---

## 5. 解析输出：把 AI 的废话变成代码变量

1. **提取核心字段**：大模型通常会啰嗦一大堆分析，我们需要用正则表达式，或者直接用大模型的 **Function Calling（工具调用）** 功能，强行让它输出一个干净的 JSON，比如 `{"action": "BUY", "confidence": 0.8, "reason": "..."}`。
2. **标准化动作**：把大模型五花八门的词，强行映射成代码认识的枚举值：强推买入（Buy）、持有（Hold）、卖出（Sell）。
3. **兜底逻辑（Fallback）**：如果大模型抽风了，没告诉你止损价在哪，代码里必须有个 `if` 逻辑，用简单的数学公式（比如当前价格跌 3%）自动补上止损价。

---

## 6. 最后一道防线：代码级风控（绝对不让 AI 碰）

1. **本地脚本拦截**：拿着大模型给出的建议（比如建议买入 50% 仓位），跑一遍本地的 Python 风控函数。如果发现它违背了第一步设定的红线（比如没钱了、仓位超了），直接把这个建议 `Drop` 掉。
2. **算一下该买多少**：看看大模型给的信心指数（`confidence`），信心高就多买点，信心低就少买点（可以套用凯利公式等量化算法）。

---

## 7. 执行交易：调 API 下单

1. **发请求**：一切校验通过后，调用券商的下单 API（如 `submit_order` 或 `place_order`），把大模型的决定变成真正的买卖动作。
2. **监听结果**：用死循环或 WebSocket 盯着这个订单，看看是已成交（Filled）还是被撤单（Canceled）了。
3. **复盘记账**：把这次交易的分析逻辑、买入价格存进数据库。等以后平仓了，再把赚了还是亏了更新进去，留着给以后的大模型当“长记仇/学习用。
