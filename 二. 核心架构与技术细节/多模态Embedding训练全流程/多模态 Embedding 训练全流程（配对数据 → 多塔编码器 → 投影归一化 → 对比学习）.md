# 多模态 Embedding 训练流程（多模态数据 → 算子级链路 → 统一向量空间）

目标：从“多模态配对数据”开始，按一次训练作业的真实计算顺序，串起数据进 batch、各模态塔的 token/patch/帧级算子、投影与归一化、跨卡负例、对比学习 loss、反向更新与保存产物，直到得到一个可上线检索的 embedding 模型。每一步都同时回答“做了什么/为什么这么做”，并配一句“就像……”的类比。

---

## 1. 数据进入训练作业（样本定义 → 采样 → batch 结构）

这一步的产物是：一个 batch 的“多模态输入张量” + “正样本对应关系” + “样本权重/掩码”。

1. 样本契约：把训练数据抽象为 `D = {(x_i^a, x_i^b, meta_i)}`，`a/b` 是 text/image/audio/video/doc（为了明确训练信号来自“谁和谁是正样本”；就像先把成对的钥匙和锁放进同一个袋子并贴好标签）
2. 采样与对齐：从 `D` 采样 `B` 条，得到 `pairs = {(x_i^a, x_i^b)}`，并保留 `pos_index(i)=i` 或多正样本索引集合 `P(i)`（为了让一次前向里同时出现大量负例且不把真相关当负例；就像把同一场考试的“正确答案索引”先写在答题卡上）
3. Batch 形状（示例）：文本侧 `input_ids: [B, T]`，视觉侧 `pixels: [B, C, H, W]`（或视频 `frames: [B, F, C, H, W]`），音频侧 `wave: [B, L]`（为了在 GPU 上并行计算；就像把不同格式的材料装进标准化托盘送上流水线）
4. 权重与掩码：从 `meta_i` 派生 `sample_weight: [B]`、`language_bucket`、`is_noisy` 等（为了把“数据质量/分布控制”落实到 loss 权重与评测分桶；就像给每件材料贴上质检等级，决定后续如何计分）

---

## 2. 模态内预处理与张量化（CPU/I/O → GPU 可算输入）

这一步的产物是：每种模态的“模型可吃”的数值张量（包含 padding/mask）。

1. 文本侧算子：分词 `tokens → input_ids` + padding 生成 `attn_mask`（为了把字符串变成可查表的离散索引，并避免 padding 污染统计；就像把句子拆成统一规格的“零件编号”，空位用隔板隔开）
2. 图像侧算子：解码/resize/normalize，得到 `pixels`（可选 augmentation）（为了把不同分辨率与色彩分布统一到模型期望的输入域；就像把照片冲洗成统一尺寸再进相册）
3. 视频侧算子（如有）：抽帧/对齐 `F` 帧，得到 `frames`（为了把“时间维”显式变成可批处理的张量维度；就像把一段视频剪成固定数量的关键帧卡片）
4. 音频侧算子（如有）：裁剪对齐长度 `L`，或做 STFT/梅尔谱得到 `spec: [B, Freq, Time]`（为了把波形变成更容易学习的时频结构；就像把声音改写成可读的五线谱）

---

## 3. 多塔前向：从 token/patch/帧到隐藏表征（Encoder 级算子链）

这一步的产物是：每个样本在各自模态塔的隐藏表征序列与聚合向量。

1. 文本塔输入嵌入：`X_t = TokEmbed(input_ids) + PosEnc`，形状 `[B, T, d_t]`（为了把离散 id 映射到连续空间并注入顺序信息；就像把编号换成坐标并标出座位号）
2. 文本塔 Encoder：`H_t = TextEncoder(X_t, attn_mask)`（Transformer/Conformer 等），输出 `[B, T, d_t]`（为了让 token 间信息交互形成语义上下文；就像让一段话内部互相“串联解释”）
3. 视觉塔 patchify：把 `pixels` 切成 patch 并线性映射：`X_v = PatchEmbed(pixels)`，形状 `[B, N, d_v]`（为了把像素转成一串可序列建模的“视觉 token”；就像把照片切成小拼图块并编号）
4. 视觉塔 Encoder：`H_v = VisionEncoder(X_v)`，输出 `[B, N, d_v]`（为了把局部 patch 组合成全局语义；就像把拼图块拼成场景理解）
5. 序列到单向量聚合：`h_i^m = Pool(H_m[i, :, :])`（如 [CLS]/mean/attn pool），得到 `h^t: [B, d_t]`、`h^v: [B, d_v]`…（为了让检索库能用“一条数据一个向量”做索引；就像把一本书压缩成一张摘要卡）

---

## 4. 投影头与归一化（对齐维度 → 统一相似度几何）

这一步的产物是：统一维度、可直接做相似度的 embedding `z`。

1. 线性/MLP 投影：`u^t = Proj_t(h^t)`，`u^v = Proj_v(h^v)`，得到 `u^*: [B, d_embed]`（为了把不同塔输出映射到同一维度空间；就像把英寸/厘米都换算成米）
2. L2 归一化：`z = u / ||u||_2`，得到 `z^*: [B, d_embed]`（为了让 cosine/点积在几何上等价并减少“向量长度抢戏”；就像把音量统一到同一档再比较旋律像不像）
3. 温度缩放：设 `τ`（固定或可学习），后续相似度用 `/ τ`（为了控制 logit 尺度与梯度稳定性；就像调节“评分放大器”的增益）

---

## 5. 跨模态相似度矩阵（GEMM 级主算子）

这一步的产物是：一个 batch 内的相似度矩阵 `S`（以及分布式下的更大负例池）。

1. In-batch 相似度：以 text↔image 为例，`S = (z^t (z^v)^T) / τ`，形状 `[B, B]`（本质是一次矩阵乘法 GEMM）（为了把“谁像谁”变成一张可优化的打分表；就像把每个考生对每个选项的匹配度一次算成表格）
2. 多方向对齐：需要双向检索时同时计算 `S_t2v` 与 `S_v2t = S^T`（为了让两个方向都学到“把真配对排第一”；就像既训练“题 → 答案”，也训练“答案 → 题”）
3. 多卡负例池（分布式常见）：对被检索侧向量做 `AllGather`，如 `z^v_all = AllGather(z^v)` 得到 `[B*W, d_embed]`，再算 `S_t2v: [B, B*W]`（为了让有效负例数量随卡数扩大；就像把多个考场的候选答案合并成一个大题库）

---

## 6. 对比学习 loss（softmax / sigmoid，把排序目标写成标量）

这一步的产物是：一个标量 loss（以及可监控的对齐统计）。

1. InfoNCE（softmax）text→image：`p(j|i) = softmax(S[i, :])`，`L_t2v = - (1/B) * Σ_i log Σ_{j∈P(i)} p(j|i)`（为了让正样本概率最大、其他自然成为负例；就像要求正确答案在候选列表里排名最高）
2. 对称训练：`L = (L_t2v + L_v2t) / 2`（为了避免只学单向映射导致空间几何偏置；就像双向翻译更不容易“只会英译中不会中译英”）
3. Sigmoid / pairwise（可选）：对每对 `(i,j)` 用二分类 logit loss（为了在多正样本/噪声较大/批大小受限时更稳；就像逐题判断“这对应该更像还是更不像”）
4. 假负例防护：对可能相关但未标注的样本降权/屏蔽，或并入 `P(i)`（为了避免把真实相关推远导致召回下降；就像别把另一种正确解当成错题扣分）

---

## 7. 反向传播与参数更新（从 loss 到新模型）

这一步的产物是：更新后的多塔编码器参数与投影头参数（以及可复现的训练状态）。

1. 反向：`loss.backward()`（自动微分沿投影、归一化、Encoder 层层回传梯度）（为了把“排序做错了多少”转换成每个权重该怎么改；就像根据错题把每个知识点的薄弱环节标出来）
2. 数值稳定（工程常见）：混合精度、梯度裁剪、loss scaling（为了避免溢出/梯度爆炸让训练翻车；就像高速行驶加稳定器，防止抖动越放越大）
3. 优化器 step：AdamW/SGD + warmup/decay 等（为了在收敛速度与泛化之间取得平衡；就像调节学习进度：先热身再按计划加量）
4. 训练态保存：保存 `model_state`、`optimizer_state`、`scaler_state`、随机种子与数据版本（为了断点续训与复现同一空间；就像把配方、火候和原料批次都记在案）

---

## 8. 训练产物：从“训练中的塔”到“可用的 embedding 模型”

这一步的产物是：线上可调用的 embedding 推理图与明确的向量契约。

1. 导出推理路径：保留 `TextEncoder/VisionEncoder/... + Proj + Normalize`（为了线上生成向量时与训练目标完全一致；就像出厂的量尺必须和车间里用的一样刻度）
2. 向量契约固化：明确 `d_embed`、是否 L2、线上相似度用 cosine/点积、温度是否仅训练期使用（为了避免“训练空间”和“线上检索距离”定义不一致；就像比赛用尺和训练用尺不能不一样）
3. 向量版本化：为模型与向量打版本（必要时重建索引）（为了避免新旧向量不可比导致检索退化；就像换了坐标系就得重画地图）

---

## 9. 离线评测与空间诊断（不只看 loss，要看几何与检索）

这一步的产物是：可对比的离线指标、失败样例与下一轮迭代方向。

1. 双向检索评测：在独立测试集做 text→image / image→text，报告 Recall@K / nDCG@K（为了直接测“能不能把真配对排前面”；就像用标准化实战题检验）
2. 分布与塌缩诊断：看 `S` 的分布、正负间隔、跨语言/跨域分桶差异（为了发现“loss 看似正常但空间几何坏掉”；就像体检不仅看体重，还看血脂心肺）
3. 失败样例回归：把 Top-K 误召回按原因分桶沉淀回归集（为了让迭代可追踪、可复盘；就像把错题本变成训练计划）

---

## 10. 上线链路（embedding 生成 → ANN 索引 → 监控闭环）

这一步的产物是：可用的检索系统形态（模型 + 向量库 + 指标闭环）。

1. 向量化策略：库侧离线 embed、查询侧实时 embed，使用同一模型版本（为了在成本与时延间平衡；就像把大件家具先装好放仓库，小件现场快速装配）
2. ANN 索引与压缩：HNSW/IVF/量化等，权衡召回-延迟-内存（为了在毫秒级延迟里拿到足够召回；就像在有限车道里规划又快又不堵的路线）
3. 监控与回滚：监控分布漂移、召回/点击/满意度变化，保留回滚路径（为了防止换模型导致全站检索“变味”；就像升级导航算法先灰度并留后路）
4. 线上评估：A/B 或灰度对比不同模型/索引配置，自动回收失败样例（为了闭环验证“离线好”是否等于“线上好”；就像实验室测得再好也要上路试驾）
