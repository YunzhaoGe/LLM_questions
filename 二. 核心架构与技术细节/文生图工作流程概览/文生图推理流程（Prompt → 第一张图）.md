# 文生图推理流程（Prompt → 第一张图）

目标：从用户输入 prompt 开始，串起现代文生图系统的“条件编码 → 连续时间采样（扩散/流）→ 去噪器（更常见为 Transformer/Diffusion Transformer）→ 解码成 RGB”的主链路。尽量用当前主流工程实现的表述：时间步通常是连续时间/噪声强度（sigma），推理也常结合少步采样与蒸馏/一致性加速；去噪器输出可能是 ε/v/x0 中的一种或多种。并且每一步都回答“做了什么/为什么这么做”，配一句“就像……”的类比，帮助建立直观的全局理解。

---

## 1. 文本输入管线（Prompt → Token IDs）

1. Prompt 规范化（可选）：清理空白、权重语法、模板化（为了减少“同义不同写法”带来的不稳定；就像把需求写成统一格式的工单）
2. Text Tokenization：把 prompt 文本切分为 token 序列（为了把自然语言变成模型可处理的离散符号；就像把一句话拆成可复用的“积木块”）
3. Vocabulary Lookup：把 token 映射成 token_id（为了用整数编号快速查表；就像把词换成字典页码，查得更快）

---

## 2. 语义条件构建（Text/Multimodal Conditioning）

1. Text Encoder Forward Pass（如 T5/CLIP/多编码器）：把 token 序列编码成条件表征（token-level 条件 + pooled 条件等）（为了把“人类语言”变成“模型可对齐的语义条件”；就像把需求说明书转成机器可执行的规格）
2. Negative / Unconditional Conditioning（可选，CFG 常用）：再编码一份负面或无条件条件（为了在生成时做“想要/不要”的对比控制；就像同时给出正面需求和避坑清单）
3. 多模态条件（可选，现代工程常见）：额外输入参考图/姿态/深度/边缘/风格等条件（为了把“文本以外的约束”也纳入生成；就像给设计师附上参考图和线框图）

---

## 3. 初始状态设定（Latent & Schedule）

1. Latent 初始化：在潜空间采样初始噪声（由 seed 决定，可复现）（为了从“随机起点”开始逐步收敛到目标图像；就像先有一团噪点，再慢慢擦出轮廓）
2. 尺寸与潜空间比例：确定输出分辨率与对应的 latent 尺寸（为了让模型知道要“画多大”；就像先选画布尺寸）
3. 连续时间/噪声强度日程：确定一串 `sigma(t)`（或连续时间点）与步数 N（现代推理常追求更少步数，常结合蒸馏/一致性加速）（为了规定“每一步去多少噪、走多大步”；就像规定打磨从粗砂到细砂的节奏）
4. 选择求解器/采样器：确定一步更新规则与超参（如 Euler/Heun/DPM-Solver++ 等家族）（为了用更少步数更稳定地收敛；就像选一条更顺的路线开到目的地）

---

## 4. 每步输入组装（Guidance & Prepare）

对每个采样步 `k = 1..N`：

1. Guidance 参数（可选）：CFG scale、rescale、提示词权重等（为了在“更听 prompt”与“更自然”之间做可控折中；就像调节‘严格按需求执行’的程度）
2. 取当前 sigma/时间：得到本步的 `sigma_k`（或 `t_k`）（为了让模型知道“现在噪声有多大”；就像告诉修图师当前有多少雾需要处理）
3. Latent 预处理：按采样器定义对 latent 做缩放/归一化（为了让不同 sigma 下的数值尺度稳定；就像不同阶段用不同力度打磨）
4. 组装条件：准备条件 token、pooled 条件，以及（可选）负面条件（为了让每一步都在同一份“需求约束”下去噪；就像每次改稿都对照同一份需求文档）

---

## 5. 去噪器推理（Denoiser Forward）

1. 去噪器输入：把 latent + sigma/time embedding + conditioning 一起送入去噪器（若使用 CFG，常将条件/无条件拼 batch 一次前向得到两路输出）（为了让模型同时知道“我在什么阶段”以及“要画什么”；就像给画师同时报进度和给参考要求）
2. 条件注入机制：在网络内部用 cross-attention、调制（AdaLN/FiLM 类）等方式融合条件（为了让文本/参考条件真正影响生成；就像在画画过程中反复对照参考图与描述）
3. 输出预测：去噪器输出 ε/v/x0（或它们的组合/等价参数化）（为了给求解器提供“下一步该往哪里走”的方向场；就像告诉你下一刀该怎么修才能更接近目标）

---

## 6. 求解器更新与解码（Solver Step → RGB）

1. Guidance 合成（可选）：把条件/无条件两路输出按比例合成（CFG/rescale 等）（为了增强对 prompt 的遵从并抑制跑偏；就像把“按需求”与“自由发挥”做加权折中）
2. 求解器一步更新：用 ODE/SDE 求解器把 latent 从 `sigma_k` 更新到 `sigma_{k+1}`（为了把噪声逐步消掉并形成结构；就像按一套固定手法反复擦拭，雾越来越薄）
3. 循环：回到下一步，直到 `sigma` 接近 0（为了收敛到可解码的潜表示；就像多轮迭代把粗糙变精致）
4. Latent 后处理（取决于实现）：把最终 latent 调整到解码器期望范围（为了避免颜色/对比度异常；就像出片前先把曝光白平衡校准到合理范围）
5. Decoder：用解码器（常见为 VAE 或其它图像解码器）把 latent 还原成 RGB 图像（为了把“潜空间的压缩表示”变成肉眼可见的像素；就像把压缩包解压成可直接查看的图片）

---

## 7. 交付与增强（Post & Output）

1. 细化/放大：超分、细节增强、分块重绘等（为了提升清晰度与局部细节；就像在成片上做精修）
2. 安全与水印：内容过滤、标识/水印（为了满足合规与溯源；就像发布前的内容审核与盖章）
3. 保存与返回：输出图片与元信息（seed、步数、采样器等）（为了复现与追溯；就像把配方和工艺也一起存档）
