# 多模态大模型视觉理解流程（图片 → 自然语言）

把“图片输入”如何一路变成“可被语言模型处理并联合推理的文本序列”讲清楚：图像张量解析 → 预处理与标准化 → 分块与嵌入 → 视觉特征提取 → 语义投影对齐 → 跨模态序列融合 → 联合推理与解码。

---

## 1. 图像输入与数组解析（Raw Pixel Input）

1. 读取像素数据：模型拿到的不是“图片文件”，而是一块由高度、宽度和颜色通道（RGB）组成的三维数组，也常叫张量（Tensor）（为了把物理图片变成程序能处理的数值输入，每个像素值通常在 0~255 之间）
2. 明确输入规模：根据图片分辨率确定这块数组有多大，例如一张 1024×1024 的图片大约包含 300 万个数值（为了理解后面为什么要先缩放、再压缩，而不是直接把原图完整塞进模型）

---

## 2. 图像预处理（Preprocessing）

1. 统一图片尺寸（Resize）：将原始图片缩放或裁剪到模型规定的标准大小，如 448×448（为了满足视觉编码器固定的输入规格）。例如，用户上传一张 1024×768 的照片，模型不会直接原样处理，而是先缩放到统一尺寸再进入后续流程
2. 归一化（Normalization）：把像素值从 0~255 变成均值为 0、方差为 1，或映射到特定范围（为了让神经网络更容易处理这些数值，避免不同图片的亮度和对比度差异过大影响计算）。例如，一个像素点原本是 255，先除以 255 变成 1.0；如果继续按均值 0.5、标准差 0.5 做标准化，就会变成 `(1.0 - 0.5) / 0.5 = 1.0`
3. 数据增强（可选）：在训练阶段可能会随机裁剪、翻转图片，但推理阶段通常不会这样做（为了提升模型对形变、角度变化的泛化能力）

---

## 3. 网格化与展平向量化（Patch Embedding）

1. 切成小块（Patching）：把图片切成很多固定大小的小方块，比如 14×14 或 16×16 像素一个 Patch（为了把整张图改写成一串可处理的局部片段）。例如，一张 448×448 的图，如果按 14×14 切分，就会得到 32×32 = 1024 个 Patch
2. 变成向量（Flatten & Project）：把每个 Patch 的像素值展开，再通过一个线性层映射成固定长度的向量（为了把原始像素块转换成统一格式的视觉 Token）。例如，一个 14×14×3 的 Patch 原本有 588 个数，经过线性层后可以变成一个 1024 维向量
3. 补上位置信息（Positional Encoding）：给每个 Patch 额外加入位置编码，告诉模型它原本在图片的什么位置（为了避免模型只看到内容，却不知道这些内容原来位于左上角还是右下角）。例如，两个外观相似的蓝色区域，一个在天空位置，一个在衣服位置，位置编码能帮助模型区分它们

---

## 4. 视觉特征提取（Vision Encoder）

1. 送入视觉编码器：将这些带位置信息的视觉 Token 输入到专门处理图像的 Transformer 网络里，比如 ViT 及其变体（为了开始做真正的视觉理解）
2. 建立全局关系：通过自注意力机制，让每个 Patch 都能参考其他 Patch 的信息（为了让模型不只看局部纹理，还能理解“这些局部拼起来是什么”）。例如，一个 Patch 里只看到尖耳朵，单独看很难判断，但结合旁边的胡须、眼睛和毛发后，模型更容易判断这是“猫”
3. 提取高层语义：经过多层计算后，模型会把低层的边缘、颜色、纹理，逐步提炼成更高层的物体和场景特征（为了得到可供语言模型使用的视觉语义表示）

---

## 5. 视觉-文本对齐（Projection / Alignment）

1. 先做维度转换：视觉编码器产出的向量长度，通常和 LLM 文本 Token 的向量长度不同，所以要经过一个投影层把两边变成同样的维度（为了让视觉 Token 和文本 Token 可以放进同一套 Transformer 里）。例如，视觉侧可能输出 1024 维向量，而 LLM 使用 4096 维隐藏状态，那么投影层就负责把 1024 维映射成 4096 维
2. 再做语义对齐：关键不只是“长度一样”，而是投影层在海量图文配对训练中，被反向传播不断调整参数，最后学会把“猫的视觉特征”映射到接近“cat”这个文本概念的位置（为了让图片信息进入 LLM 后不只是可拼接，而且可理解）
3. 特定架构可绕过投影：少数模型会直接设计视觉模块，使其输出天然兼容 LLM 的 Token 表示（为了减少中间转换层带来的信息损耗）

---

## 6. 多模态数组融合（Multimodal Fusion）

1. 处理文本提示：如果用户同时输入了文字，比如“描述这张图”，这段文字会先被 tokenizer 切成文本 Token（为了把自然语言也转成模型内部统一的表示）
2. 拼接图文序列：把对齐后的视觉 Token 和文本 Token 按顺序拼在一起，可以粗略理解成 `[...ImageTokens, ...TextTokens]`（为了构造一条同时包含图片信息和文字指令的输入序列）。例如，序列前半段是图片对应的视觉 Token，后半段是“这张图里有什么？”这句提示词对应的文本 Token
3. 输入主模型：把这条混合序列整体送进 LLM 主干网络（为了让后续的语言推理同时参考图片和文本）

---

## 7. 联合推理与解码（Joint Reasoning）

1. 做跨模态注意力计算：进入 LLM 后，图像 Token 和文本 Token 会一起参与自注意力计算（为了让模型把文字里的“狗”和图片里的那只狗对应起来）
2. 计算候选词分数（Logits）：模型基于当前上下文，给词表里的每个候选词打分（为了决定下一个最可能输出的词是什么）。例如，如果图里是一只猫，那么“猫”“小猫”“动物”这些词的分数通常会比“汽车”“飞机”更高
3. 逐个生成文本：模型会以自回归方式一个 Token 一个 Token 往外生成，过程可以类比成一个不断追加输出的循环（为了最终产出描述、回答或其他文本结果）

---

## 8. 核心训练机制与本质

1. 先学图文对应：模型通常先在海量“图片 + 文本描述”数据上训练，让相关图片和文字在表示空间里逐渐靠近（为了建立基础的图文对应关系）
2. 再学按指令回答：然后再用“图片 + 问题 → 答案”这类数据继续训练（为了让模型不仅能认图，还能按用户要求组织输出）
3. 本质仍是模式匹配：模型并不是真的像人类一样“看见”了图片，而是在高维向量空间里学会了稳定的图文对应模式（为了在工程上实现接近语义理解的效果）
