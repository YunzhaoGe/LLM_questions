# LoRA 低秩适配微调流程（底座权重 → 低秩增量 → 训练更新 → 导出适配器）

目标：从一个预训练底座模型 checkpoint 开始，串起“决定改哪些层 → 在权重旁边挂低秩增量 → 用数据做前向/反向 → 只更新增量参数 → 导出可加载的适配器”，直到你拿到一份可上线复用的 LoRA 权重。每一步都同时回答“做了什么/为什么这么做”，并配一句“就像……”的类比，帮助建立直观的全局理解。

---

## 1. 明确输入与目标（先对齐“我要把模型变成什么样”）

1. 底座权重：加载预训练模型 checkpoint `W_base`（为了把通用能力当作地基，后续只做定向改动；就像先拿到一台能跑的发动机，再决定加哪套改装件）
2. 数据与切分：准备训练/验证集，并做最基本的清洗与去重（为了让监督信号稳定，避免重复与污染把更新方向带偏；就像先把教材编好，删掉错题和重复题）
3. 验收口径：定义 2–3 个最关键指标或抽检维度（格式、事实性、工具调用等）（为了避免“看起来更好”但上线翻车；就像不只看作业分，还要做一次模拟考试）

---

## 2. 选定注入位置（决定“改哪里”比“怎么改”更关键）

1. 目标模块：选择要挂 LoRA 的线性层（常见是 Attention 投影与 MLP 投影）（为了把更新预算花在最能改变路由与表征的地方；就像只拧关键阀门，而不是把整条管道都换掉）
2. 覆盖范围：决定挂载层数与分布（全层/后几层/间隔挂载）（为了在效果提升与稳定性/开销之间做结构化取舍；就像只改关键路口的红绿灯，不一定要全城一起动）
3. Rank 起点：选一个保守的 `r` 作为起步（为了先验证方向对不对，再加表达力；就像先换轮胎试跑，再决定要不要换悬挂）

---

## 3. 添加低秩增量（把“大矩阵更新”改成“小模块学习”）

1. 低秩参数：为每个目标层新增 `A` 与 `B`（`A ∈ R^{r×d_in}`，`B ∈ R^{d_out×r}`）（为了把改动限制在秩不超过 `r` 的子空间；就像用两段“窄管”串起来改变流向，而不是重铺大水管）
2. 前向形态：把 `y = xW^T` 变成 `y = xW^T + scale * ((xA^T)B^T)`（为了让 LoRA 作为旁路增量加入，不破坏主干结构；就像在主流水线上加一个可插拔的小工位）
3. 缩放约定：常见 `scale = α / r`（为了让增量尺度可控，并让不同 rank 下更好复用经验超参；就像给改装件加限幅器，防止一上来就推爆输出）

---

## 4. 冻结底座（训练时只让“增量”动）

1. 冻结 `W`：底座权重不参与更新（为了省显存与优化器状态，并降低破坏通用能力的风险；就像把地基封死不动，只在上面加可拆卸模块）
2. 初始化：让 `ΔW` 初始近似 0（常见 `B=0`）（为了让训练起点几乎等同底座，优化更稳；就像改装先从怠速试起）
3. 参数分组：只把 LoRA（与少量解冻参数）交给优化器（为了把最贵的动量/方差状态也压到最小；就像只给改装件配工具箱，不给整台机器每个零件都配一套）

---

## 5. 把样本变成训练张量（让模型“吃得下、算得快”）

1. 模板与分词：把样本按 chat template 拼接并 tokenization 得到 `input_ids`（为了让训练输入与线上对齐；就像先把对话按统一版式排好再开工）
2. Labels：构造 `labels` 并明确哪些 token 参与 loss（为了让模型只为该学的部分负责；就像把标准答案区用荧光笔标出来）
3. Batch 对齐：做 padding/截断/滑窗等（为了并行训练且不让 padding 影响损失；就像把不同长度的卷子装进同样大小的文件夹，但用隔板标记空位）

---

## 6. 前向计算（底座先理解，LoRA 再偏转）

1. 主干前向：序列经过 Transformer 得到隐藏状态（为了让底座能力先“读懂上下文”；就像老厨师先把火候起好）
2. LoRA 叠加：在被注入的线性层处走低秩旁路并叠加输出（为了把定制化改动写进关键表示变换；就像在关键工位加一小勺秘制酱料）
3. 得到 logits：输出头产生每个位置的 `logits`（为了把“当前偏好哪个 token”显式化；就像把候选答案排行榜打印出来）

---

## 7. 计算 loss（把“偏好”变成可优化的数）

1. 监督损失：用 `labels` 做交叉熵/序列损失（为了把训练目标变成可微标量；就像把主观好坏变成一张可打分的试卷）
2. 重点加权（可选）：对关键字段/格式片段加权（为了让训练信号更对准产品目标；就像重点题加权计分）
3. 监控项：记录 loss 与关键指标（为了尽早发现学歪/不收敛；就像边训练边看心率与配速，避免越练越伤）

---

## 8. 反向传播（梯度只流向 LoRA）

1. 反向传播：对 loss 执行 `backward()`（为了把“哪里错了”转换成参数更新方向；就像从总分反推每道题该怎么纠错）
2. 梯度流向：由于底座冻结，梯度主要落在 `A/B`（为了把训练成本集中在小参数集合；就像只训练外挂技能包，不重塑基本记忆）
3. 稳定措施：按需做梯度裁剪/混合精度溢出处理（为了避免偶发大梯度把更新推飞；就像给方向盘加阻尼，防止高速时小抖动放大成甩尾）

---

## 9. 参数更新与训练循环（把“学到的东西”写进增量权重）

1. Optimizer step：更新 LoRA 参数（为了让增量逐步吸收数据梯度信号；就像每次试车后微调改装件角度）
2. 学习率策略：warmup + 衰减（常见）（为了训练初期更稳、后期更易收敛；就像先热身再冲刺，最后做收操）
3. 验证与早停：定期在验证集/回归集上抽检并早停（为了避免过拟合与副作用累积；就像发现成绩开始虚高就停下来复盘）

---

## 10. 导出适配器（把训练成果变成可上线的“插件/快照”）

1. 保存产物：只保存 LoRA 的 `A/B` 与注入配置（层名、rank、α）（为了体积小、可复用；就像只交付改装件和安装说明，不把整车再发一遍）
2. 绑定版本：记录底座 checkpoint 与 tokenizer/模板版本（为了避免“能加载但不对”的错位；就像手机壳必须绑定具体机型，差一代就扣不上）
3. 合并选择：动态不合并（易切换）或离线合并（专用快照）（为了在灵活性与延迟之间选工程形态；就像滤镜随时可换，或把滤镜烘焙进照片）

一句话总结：LoRA 微调就是把“需要的改动”压缩成一个可训练、可保存、可插拔的低秩增量模块，用最小的参数与状态成本，完成对底座能力的定向偏转。
