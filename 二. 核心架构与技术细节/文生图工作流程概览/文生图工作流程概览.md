1. Text Tokenization（将输入提示词文本拆分成离散的 token 序列，为了将自然语言转换为模型可处理的标准化单元，避免歧义并便于后续嵌入处理）。

2. Vocabulary Lookup（在预训练的文本词汇表中查找每个 token 对应的唯一 ID，为了将 token 映射到数值索引，便于文本嵌入层的快速检索）。

3. Text Embedding（将每个 token ID 转换为高维空间的向量表示，为了捕捉提示词的语义、语法和上下文信息，使模型能理解用户意图并生成相关图像）。

4. Positional Encoding（如 RoPE 或 Sinusoidal：将位置嵌入添加到文本嵌入向量中，为了注入序列中 token 的位置信息，让模型区分相同 token 在不同位置的含义，避免顺序混淆）。

5. Text Encoder Forward Pass（通过 Transformer 编码器（如 CLIP）处理嵌入序列，为了提取多层语义特征，生成固定长度的条件嵌入向量，用于指导图像生成）。

6. Final Text Conditioning（对编码器输出进行池化或投影，为了压缩成单一的条件向量或嵌入矩阵，便于与噪声或 latent 空间整合）。

7. Latent Space Initialization（生成随机高斯噪声作为初始 latent 表示，为了模拟扩散过程的起点，提供一个无结构的基础供模型逐步精炼成图像）。

8. Timestep Scheduling（定义去噪过程的多个时间步序列，如从 T=1000 到 1，为了模拟逆向扩散过程，逐步从噪声恢复出结构化的图像表示）。

9. Timestep Embedding（为当前时间步生成嵌入向量，如使用 Sinusoidal 位置编码，为了注入时间信息，让模型知道当前去噪阶段，便于预测噪声水平）。

10. Concatenate Conditioning（将文本条件嵌入与当前 latent 噪声和时间步嵌入拼接或融合，为了将用户提示融入到每个去噪步骤中，确保生成内容符合描述）。

11. Input Projection（对融合后的输入进行线性变换，为了调整维度或初始化隐藏状态，适应 U-Net 或主网络的输入要求）。

12. Layer Normalization（对输入进行归一化处理，为了稳定数值分布，减少内部协变量偏移，提高模型推理的稳定性）。

13. Downsampling Blocks（通过卷积和下采样层逐步压缩 spatial 维度，为了提取多尺度特征，捕捉图像的全局和局部结构，便于高效处理高分辨率）。

14. Self-Attention in Down Blocks（在下采样块中应用自注意力机制，为了捕捉 latent 空间中像素间的长距离依赖，提高对复杂图案的建模能力）。

15. Cross-Attention with Conditioning（应用跨注意力，将条件嵌入作为 Key/Value 与 latent Query 交互，为了注入文本指导，确保生成的特征与提示词语义对齐）。

16. Residual Connections in Down Blocks（在每个下采样子块中添加残差连接，为了保留原始信息，缓解深层网络的梯度消失问题，促进稳定训练和推理）。

17. Bottleneck Processing（在网络中间的瓶颈层进行密集特征变换，为了压缩和精炼最抽象的表示，桥接下采样和上采样路径）。

18. Upsampling Blocks（通过转置卷积或上采样层逐步恢复 spatial 维度，为了从抽象特征重建高分辨率表示，生成细粒度细节）。

19. Self-Attention in Up Blocks（在上采样块中应用自注意力机制，为了整合多尺度特征，确保生成的图像连贯性和一致性）。

20. Cross-Attention in Up Blocks（再次应用跨注意力融合条件嵌入，为了在重建过程中持续强化文本指导，避免生成偏离提示）。

21. Residual Connections in Up Blocks（在上采样子块中添加残差连接，为了平滑特征流，增强模型对细节的恢复能力）。

22. Skip Connections（从下采样块跳跃连接到对应上采样块，为了提供低级特征作为指导，帮助恢复边缘、纹理等精细信息，避免信息丢失）。

23. Output Projection（通过最终卷积层投影 U-Net 输出，为了生成噪声预测或更新后的 latent 表示，匹配扩散模型的期望格式）。

24. Noise Prediction（基于 U-Net 输出计算当前时间步的噪声估计，为了模拟逆扩散过程，指导 latent 的逐步去噪）。

25. Scheduler Update（使用调度器算法如 DDIM 或 PNDM 从噪声预测更新 latent 表示，为了计算下一个时间步的 latent，逐步减少噪声水平）。

26. Denoising Loop（重复步骤 9-25，对于每个时间步，通常有 20-1000 步不等，每步逐步精炼 latent 表示，为了从纯噪声构建出结构化的图像潜在表示）。

27. Final Latent Normalization（对去噪后的最终 latent 进行归一化或缩放，为了准备进入解码器，确保数值范围适合像素重建）。

28. VAE Decoder Forward Pass（通过变分自编码器（VAE）的解码器处理 latent 表示，为了将压缩的潜在空间映射回像素空间，生成可观的 RGB 图像）。

29. Post-Processing（可选，对生成的图像应用锐化、颜色校正或上采样，为了提升视觉质量，使最终输出更逼真和吸引人）。
