# 大模型的基建全貌（聚焦输入/输出两端）

当我们把“大模型”视作一个**预测下一个 token 的系统**时，它要变成“可预期、可靠、稳定”的生产力工具，最贴近用户体感的一层往往不在训练侧，而在**输入如何被组织**、以及**输出如何被约束与交付**这两端。

本文将范围限定在：从用户发出意图开始，到模型请求被构造；以及从模型产出 token 开始，到结果可安全、可验证、可运营地交付给用户。

---

## 一、输入端基建：从“用户意图”到“高质量模型请求”

### 1) 多端接入与会话层（Conversation Gateway）

- 是什么：承接 Web/IM/IDE/语音等入口的会话服务，统一会话状态、消息协议、附件上传、长连接/重连、节流与重试。
- 角色：让“同一个智能体/助手”在不同入口有一致的行为；同时把网络抖动、重复提交、并发等工程问题挡在模型之前。
- 典型产品/项目：API Gateway（Kong / Apigee）；Envoy；Socket.IO（会话实时性常见方案）

### 2) 身份、权限与上下文注入（IAM + Context Injection）

- 是什么：用户身份认证（SSO/OAuth）、RBAC/ABAC 权限校验、租户隔离；并把用户/组织/项目等上下文以安全方式注入到提示词或检索条件中。
- 角色：决定“模型能看到什么、能做什么”，避免把大模型变成越权数据访问的放大器；也是企业落地的必要条件。
- 典型产品/项目：Okta；Auth0；HashiCorp Vault（密钥与凭据管理）

### 3) 输入预处理与规范化（Input Normalization）

- 是什么：文本清洗、语言检测、格式规整（代码块/表格）、附件解析（PDF/Word/HTML）、OCR、音频转写、图像转文字等预处理。
- 角色：把“噪声输入”变成“稳定可学习/可推理的输入”，减少模型被格式与脏数据干扰。
- 典型产品/项目：PaddleOCR；Apache Tika（文档解析）；Whisper（语音转写）

### 4) 提示词/系统指令管理（Prompt & Policy Management）

- 是什么：系统提示词、开发提示词、模板变量（上下文槽位）、版本管理、灰度发布、A/B 实验、回滚；以及策略规则（不同人群/场景走不同模板或模型）。
- 角色：把“口口相传的提示词技巧”沉淀成可复用资产，让迭代变成工程发布流程，而不是改一句话全靠运气。
- 典型产品/项目：LangSmith；PromptLayer；promptfoo（提示词回归与对比）

### 5) 上下文管理与 Token 预算（Context Orchestration）

- 是什么：对话记忆裁剪、摘要压缩、片段排序、关键信息保留策略、token 预算与动态截断；必要时引入“短期记忆/长期记忆”组件。
- 角色：在上下文窗口有限、用户对话很长时，仍能保持稳定的任务一致性与可控成本。
- 典型产品/项目：LangChain Memory（概念与实践）；LlamaIndex（上下文拼装与索引）；Redis（会话/摘要缓存常用）

### 6) 检索增强生成（RAG）：知识注入与可追溯上下文

- 是什么：文档切分（chunking）、向量化（embedding）、索引与召回（向量/关键词/混合检索）、重排（rerank）、权限过滤（ACL filter）、引用片段组装（context pack）。
- 角色：解决三类关键问题：私有知识接入、时效性、以及让回答“有证据可引用”；同时把检索的可控性引入生成过程。
- 典型产品/项目：Pinecone；Milvus；Elasticsearch/OpenSearch（混合检索常用）

### 7) 工具/函数调用注册表（Tool Registry）

- 是什么：可调用工具的描述（名称、入参 schema、权限要求、速率限制、幂等键）、以及工具的版本化发布与下线策略。
- 角色：让模型从“生成文本”升级到“调用工具做事”，同时让工具调用具备工程治理（权限、审计、变更管理）。
- 典型产品/项目：OpenAI Function Calling（范式代表）；JSON Schema；OpenAPI（工具接口描述常用）

### 8) 计划与编排（Agent Orchestration）

- 是什么：任务分解（plan）、工具选择（route）、执行器（execute）、失败重试与回退（fallback）、并发/串行策略、超时控制、幂等与去重。
- 角色：把复杂任务从“一次性长输出”变成“多步可控流程”，显著提升成功率与可维护性。
- 典型产品/项目：LangGraph；Temporal（工作流编排）；OpenAI Swarm（或同类多智能体编排思路）

### 9) 输入侧安全网关（Prompt Injection / PII / Abuse 防护）

- 是什么：越狱检测、提示词注入防护（区分指令与数据、对不可信内容打标签）、PII 检测与脱敏、恶意内容过滤、来源可信度标记。
- 角色：把高风险输入拦在“进入模型/进入工具链之前”，降低数据泄露、越权、内容合规的系统性风险。
- 典型产品/项目：AWS Bedrock Guardrails；Azure AI Content Safety；OpenAI Moderation（或同类审核能力）

### 10) 成本与性能的前置治理（Routing / Cache / Rate Limit）

- 是什么：模型路由（小模型优先、大模型兜底）、请求合并与批处理、语义缓存/结果缓存、配额与限流、热点提示词缓存。
- 角色：把可用性与成本“前置稳定化”，避免模型侧被流量尖峰击穿，也避免成本不可控。
- 典型产品/项目：Redis（缓存）；Envoy（限流/熔断）；Karpenter（弹性扩容常见）

## 二、输出端基建：从“token 输出”到“可信交付”

### 1) 解码控制与确定性（Decoding Control）

- 是什么：temperature/top-p/top-k、seed、重复惩罚、logit bias、stop sequences；以及针对不同任务的参数策略（写作/代码/抽取）。
- 角色：降低输出随机性，提升可复现性与任务一致性；尤其在生产场景中，确定性往往比“灵感”更重要。
- 典型产品/项目：各云厂商 LLM API 的解码参数；vLLM（推理引擎）；TensorRT-LLM（高性能推理）

### 2) 结构化输出与约束（Structured Output / Constrained Generation）

- 是什么：强制 JSON、JSON Schema 校验、正则/语法（grammar）约束、函数调用回包结构化；必要时引入“校验失败自动重试”。
- 角色：把“可读文本”变成“可被系统可靠消费的数据”，这是大模型工程化最关键的分水岭之一。
- 典型产品/项目：JSON Schema；OpenAI Structured Outputs（或同类能力）；Pydantic（结构校验常用）

### 3) 事实性与可追溯（Citations & Evidence Pack）

- 是什么：为回答附带引用片段、来源链接、检索命中信息；对关键结论输出“证据包”（supporting context）以便审阅与回放。
- 角色：显著降低幻觉的业务伤害；让“正确性”从不可验证变成可审计、可抽检。
- 典型产品/项目：Perplexity（引用式体验代表）；LlamaIndex（引用/来源拼装）；Elasticsearch（高亮与片段定位）

### 4) 输出侧安全与合规（Output Moderation & Redaction）

- 是什么：对模型输出做敏感信息检测（PII/密钥/内部标识）、内容合规审核、必要时重写/遮盖/拒答；并对违规风险进行分类与打点。
- 角色：把“模型偶发不当输出”变成“系统层可控的风险”，这是稳定生产必备的最后一道闸门。
- 典型产品/项目：Azure AI Content Safety；AWS Bedrock Guardrails；Presidio（PII 检测/脱敏常见）

### 5) 工具执行结果校验（Validators & Post-check）

- 是什么：对工具调用结果做一致性检查（schema/约束/业务规则）、对代码生成做编译/单测/静态检查、对查询结果做权限复核与审计记录。
- 角色：避免“模型说做了但其实没做/做错了”；让系统输出从“看起来对”升级为“验证过的对”。
- 典型产品/项目：pytest（代码校验常用）；Open Policy Agent（策略校验）；Great Expectations（数据输出质量）

### 6) 流式输出与渲染层（Streaming UX）

- 是什么：token streaming、增量渲染（代码块/表格/引用）、中途取消、追问建议、结果分段（先要点后细节）。
- 角色：提升交互体验与可用性，同时降低用户“等待不确定”的焦虑；也便于在长输出时及时止损（取消/回退）。
- 典型产品/项目：Server-Sent Events（SSE）；WebSocket；React/ProseMirror（富文本渲染常见）

### 7) 观测、审计与可回放（Tracing & Audit）

- 是什么：对每次请求记录 trace（输入摘要、检索命中、提示词版本、模型版本、token 用量、工具调用链路），并支持脱敏后的回放与排障。
- 角色：没有可观测性就没有稳定性；排查“偶发失败/幻觉/越权”的唯一可行路径就是可回放的链路证据。
- 典型产品/项目：OpenTelemetry；Prometheus + Grafana；LangSmith（链路与回放）

### 8) 反馈闭环与在线回归（Feedback Loop）

- 是什么：点赞/点踩、原因标签、人工复核队列、在线 A/B、失败样本沉淀为评测集，驱动提示词、RAG、路由与对齐数据迭代。
- 角色：把系统从“上线即定型”变成“持续变稳”，用数据把不可控逐步压缩为可控。
- 典型产品/项目：Weights & Biases（实验追踪常用）；Evidently AI（数据/模型监控思路代表）；promptfoo（回归）

## 三、一个典型端到端链路（输入到输出）

1. 用户在 Web/IDE/IM 发起请求 → 会话层归一化输入（文本/附件/OCR/转写）
2. 身份与权限校验 → 注入租户/项目上下文（可见范围、可用工具）
3. 触发策略：选择提示词版本与模型路由（小模型优先/大模型兜底）
4. 上下文编排：对话摘要 + 关键片段 + token 预算裁剪
5. RAG：检索召回 + 重排 + ACL 过滤 → 组装证据上下文
6. 工具编排：必要时生成函数调用参数 → 执行工具（带幂等/超时/重试）
7. 模型生成：解码参数控制 → 结构化输出约束
8. 输出后处理：结构校验/事实性引用/敏感信息脱敏/内容审核
9. 渲染交付：流式展示 + 可追溯引用 + 可操作的下一步建议
10. 观测与反馈：trace 入库 + 失败样本进入回归集 + 迭代提示词/RAG/路由策略
