# 分词器预训练：如何从海量文本中抽象出词表（BPE算法核心）

将海量人类文本压缩成一个固定大小的词表（Vocabulary），其核心工程实现是字节对编码（BPE, Byte Pair Encoding）的训练过程。该过程本质上是一个基于统计的贪婪压缩算法，整个训练在模型正式发布前已彻底完成。

## 1. 语料准备与预分词（Pre-tokenization）

在统计之前，必须对海量连续文本进行物理切断，防止跨越语义边界的不合理合并。
1. 强制切分规则：按空格、标点符号及特定语言规则（如正则表达式）将语料初步切分成一个个“单词”或“基础片段”。
2. 边界隔离：预分词确保了像 `dog.` 这样的组合不会在后续被合并为一个 Token，标点和字母的统计域被严格物理隔离。

## 2. 字节底座初始化（Byte-level Initialization）

统计的起点不是人类认知中的“字”或“词”，而是计算机的底层编码。
1. 降维到字节：将预分词后的所有片段转换为 UTF-8 字节序列。系统的初始基础词表仅包含 0~255 这 256 个单字节 Token。
2. 零 OOV 保证：以字节为起点虽然打碎了直观语义，但从物理层面覆盖了所有已知和未知的字符序列，彻底消除了未登录词（OOV, Out of Vocabulary）导致系统崩溃的风险。

## 3. 全局频率统计（Frequency Counting）

在基础字节序列上，展开全局扫描与频次计算。
1. 共现扫描：遍历全部语料库，统计所有“相邻两个 Token”组合出现的绝对频次。
2. 高频凸显：例如在英文语料中，字节 `t` 和字节 `h` 相邻出现的频次可能高达数亿次，位居统计榜首；而在中文语料中，编码 `中` 和 `国` 对应底层的某几个字节组合也会呈现极高的共现率。

## 4. 贪婪合并（Greedy Merging）

基于统计结果，执行最高效的词表扩充动作。
1. 提取最高频组合：取出当前频次最高的一对 Token（如 `t` 和 `h`）。
2. 创造新 Token：将它们合并为一个全新的 Token `th`，并将其加入词表。
3. 语料重写：将语料库中所有相邻的 `t` 和 `h` 替换为新 Token `th`。此时，语料的总长度变短了，但词表的总容量增加了 1。

## 5. 循环迭代与阈值熔断（Iteration & Stopping）

合并过程是一个巨大的循环操作，通过算力硬推完成。
1. 递归生长：下一轮统计会在已包含 `th` 的新语料上进行。随后 `th` 和 `e` 可能成为最高频对，合并产生新的 Token `the`。
2. 停止条件：循环会一直持续，直到词表达到工程师预设的固定大小（如 GPT-4 的 100,277，或 GLM-4 的 151,329），算法强制终止。

## 6. 最终产物固化（Final Output）

训练结束后，丢弃原始海量语料，仅保留两个核心资产文件：
1. 词表映射（Vocab）：记录所有 256 个基础字节 Token 以及历次合并产生的新 Token，并为每个 Token 分配唯一整数 ID。
2. 合并规则（Merges）：严格记录了从初始状态到最终状态的每一次合并动作及其优先级顺序。这套规则将指导未来的线上系统，如何按部就班地拆解和合并全新的用户输入。
