# LLM 推理流程（Prompt → 第一个 Token）

把“用户输入”如何一路变成“第一个输出 token”讲清楚：输入组装 → 分词编号 → 向量与位置 → prefill 前向（含因果约束与 KV cache）→ logits → 解码决策。

---

## 1. Tokenization：文本到数字编号

1. 分词：用分词器（tokenizer）把输入文本切分成文本片段序列（为了把字符串映射到模型词表空间）
2. 编号：将文本片段转化为对应的数字编号，并记录序列的有效长度（为了让后续张量与掩码可计算）
3. 构造批次：对齐多个序列的长度（补齐到最大长度），组合成批次矩阵（为了并行推理）

---

## 2. 掩码（Mask）与位置信息准备

1. 注意力掩码：标记出为了对齐长度而填充的空白位置（为了避免把填充物当真实内容参与注意力计算）
2. 因果掩码：生成自回归屏蔽矩阵（为了避免当前位置看到未来的文本内容）
3. 位置参数：准备位置编号或旋转位置编码（RoPE）等所需参数（为了让注意力机制区分词汇的顺序与距离）

---

## 3. 词嵌入（Embedding）与张量就绪

1. 迁移到设备：将数字编号、掩码和位置参数迁移到推理设备并转换数据类型（为了匹配推理引擎执行环境）
2. 查表向量化：通过词表嵌入矩阵，将每个数字编号映射为高维连续特征向量（为了进入连续向量空间）
3. 初始化残差：将当前的特征向量设为初始残差状态，并准备进入第一层神经网络（为了配合残差结构的逐层更新）

---

## 4. 注意力（Attention）子层（单层内）

1. 归一化与线性投影：对输入特征进行层归一化后，分别经过线性变换生成查询向量（Query）、键向量（Key）和值向量（Value），并拆分为多个注意力头（为了产生注意力计算所需的查询与键值）
2. 位置注入与打分：将位置信息融入查询与键向量中，计算它们之间的点积并根据注意力头的维度进行缩放，得到原始的注意力分数（为了得到可比较的注意力分数）
3. 加权汇总：对注意力分数应用掩码和偏置后，通过 softmax 函数转化为注意力权重，再用该权重对值向量进行加权求和；最后合并所有注意力头的结果，经过输出投影矩阵，并加上此前的残差连接（为了得到注意力输出并更新特征）

---

## 5. 前馈网络（MLP/MoE）子层（单层内）

1. 归一化：对上一子层的输出再次进行层归一化（为了进入前馈子层的稳定输入）
2. 变换：执行前馈网络的上投影操作，通过激活函数或门控机制（如果是混合专家 MoE 架构则进行路由分配和专家计算），将特征映射到更高维度（为了做逐词汇的非线性通道混合）
3. 回投影：通过下投影（或专家结果聚合）将特征降维回模型的标准隐藏层维度，并再次叠加残差连接（为了完成该层的输出更新）

---

## 6. 键值缓存（KV cache）与跨层堆叠

1. 写入缓存：将当前层计算出的键向量和值向量（已包含位置信息）保存或追加到键值缓存中（为了后续解码阶段复用历史信息）
2. 重复多层：按上述流程逐层前向计算，直到穿过所有 Transformer 层，得到最终的深层特征表示（为了完成 prompt 处理的整段前向计算）
3. 得到产物：输出本次请求更新后的键值缓存以及最后一层的特征表示（为了支撑后续提取最后一个位置的信息并启动解码）

---

## 7. 取最后位置：从序列到单向量

1. 选择位置：根据每条样本的有效长度，提取出序列中最后一个有效词汇（即刚读完 prompt 的那个位置）对应的特征向量（为了取“读完 prompt 后”的当前状态）
2. 最终归一化：对提取出的最后一个特征向量进行最终的层归一化（为了匹配输出头的输入分布）
3. 进入输出头：将归一化后的向量输入到语言模型输出头（LM Head）进行线性变换（为了得到词表维度的打分）

---

## 8. 得分（Logits）处理与第一个 token 选取

1. 得到初始得分：通过输出头计算出词表中每个词汇成为下一个输出词汇的候选得分（为了得到下一 token 的候选得分）
2. 调整得分：应用温度系数、重复惩罚、坏词屏蔽、偏置或语法约束等规则调整候选得分（为了把输出限制在产品目标与格式边界内）
3. 选取并反分词：通过贪心策略或随机采样选出下一个词汇的数字编号，再将其解码为人类可读的可见文本（为了产出用户看到的首段输出）
